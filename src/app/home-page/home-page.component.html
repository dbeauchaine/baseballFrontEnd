Dan's Demonstration Web App
Overview

This website is an example project to serve as a demonstration of my software development ability. It is a work in progress and by no means does it represent a finished product. Please contact me at danbeauchaine@gmail.com with questions or comments.

Individual player statistics can be accessed by searching for their name (partial searches work) in the search bar at the top right, direct naviagion using their Lahman ID, or by clicking their name in a leaderboard. An example of an individual player's statistics: Ichiro Stats.

Leaderboards can be accessed via the top bar, or by direct navigation using the desired year. An example of a batting leaderboard: 2000 Batting Leaderboard.

Individual team statistics can be accessed by clicking on a team in the list of teams, direct navigation to /teams, by clicking on team in the team standings, or by selecting team statistics from the menu bar and selecting a team.

Team standings can be accessed by selecting standings from the team menu. Example of team standings: 1974 Team Standings.


Hosting

This website is hosted through Microsoft Azure. The database is hosted as a SQL database with a maximum of 10 DTU computing power. The back end API application is hosted as a free tier App service. The front end Angular application is hosted as a v2 storage account configured as a static website.

These hosting options keep costs low, but also pose response time issues. As a solution, I am adjusting my queries and processing to balance the computing load between the API application and the SQL database. Additionally, I am adding in-memory caching to improve query response times. A particular point of emphasis is the initial API call.


The Back End The back end is a .NET core application written in C# which serves Rest API endpoints to baseball statistics obtained from the Lahman Database and statistics computed from that data. The back end application interacts with the SQL database via Entity Framework Core. All services, controllers, mappers, and calculators are unit tested via Nunit. The back end project is hosted on github here.
The Front End

The front end is a website built with angular with services components implemented in Typescript. I have used Angular Material components when possible. The styling is achieved through sass. Components and services are unit tested using jasmine. The front end application calls the back end API and displays the data in useful tables. The front end project is hosted on github here.
